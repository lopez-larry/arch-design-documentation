# **Procurement Templates**

---

## **1) RFI – Request for Information (Template)**

```
Title: Request for Information – <Topic/Capability>
Date Issued:
Issued By:
Contact Details:

1. Background
   - Brief business context and problem space

2. Objective of RFI
   - What information we need and why

3. Requested Information from Respondents
   - Company profile
   - Technical capabilities
   - Service catalog
   - Relevant experience/case studies
   - Licensing or pricing models (high level)
   - Delivery/location constraints

4. Response Format
   - Word/PDF
   - Max length
   - Required attachments

5. Timeline
   - Questions due:
   - Responses due:
   - Next steps after review

6. Legal/Notes
   - “This RFI does not commit <Company> to any purchase.”
```

---

## **2) RFP – Request for Proposal (Template)**

```
Title: Request for Proposal – <Project/Service Name>
Date Issued:
Issued By:
Primary Contact:

1. Project Overview
2. Scope of Work
   - Deliverables
   - Client responsibilities
   - Vendor responsibilities

3. Requirements
   - Functional requirements
   - Technical environment
   - Security & compliance
   - Service level expectations

4. Proposal Requirements
   - Work plan & timeline
   - Team composition
   - Demonstrated experience
   - Assumptions & risks
   - Pricing breakdown

5. Evaluation Criteria
   - Technical fit (xx%)
   - Cost/value (xx%)
   - Timeline feasibility (xx%)
   - Vendor track record (xx%)

6. Admin
   - Deadline to submit
   - Clarification process
   - Award decision date
```

---

## **3) RFQ – Request for Quote (Template)**

```
Title: Request for Quote – <Item/Service>
Requester:
Contact:
Due Date:

1. Description of Goods/Services Requested
   - Quantities
   - Expected specifications
   - Delivery requirements

2. Vendor Response
   - Unit price
   - Total price
   - Taxes/fees
   - Shipping or support costs
   - Warranty terms

3. Validity of Quote
   - Must remain valid until <date>

4. Selection Method
   - Lowest total price
   - Value options may be reviewed
```

# **Product & Engineering Governance**

---

## **4) RFC – Request for Comments (Template)**

```
RFC Title:
RFC Number:
Author:
Date:
Status: Draft / Review / Accepted / Rejected / Superseded

1. Context
   - What triggered this proposal (problem + constraints)

2. Proposal Summary
   - Overview of recommended change
   - Diagram or example if useful

3. Alternatives Considered
   - Option A
   - Option B
   - (and why not selected)

4. Impact Analysis
   - Systems affected
   - Data or API changes
   - Risks
   - Security/compliance implications

5. Migration / Implementation Plan
   - Phases or timeline
   - Dependencies

6. Required Feedback From
   - Teams / SMEs / Architecture Council

7. Decision Record Link (ADR)
```
